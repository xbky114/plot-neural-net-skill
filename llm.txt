# PlotNeuralNet Python API

`pycore.tikzeng` and `pycore.blocks` APIs. See SKILL.md for paths and workflow.

## Document Structure Functions (tikzeng)

| Function | Description |
|----------|-------------|
| `to_head(projectpath)` | Emit `\documentclass`, `\usepackage{import}`, `\subimport{.../layers/}{init}` |
| `to_cor()` | Define color macros for Conv / Pool / Unpool / Fc / Softmax / Sum |
| `to_begin()` | `\begin{document}`, `\begin{tikzpicture}`, connection styles |
| `to_end()` | Close tikzpicture and document |

## Layer Functions (tikzeng)

Each layer's `name` becomes a TikZ node name used for connections. Anchor format: `{name}-east`, `-west`, `-north`, `-south`, `-northeast`, `-southeast`, `-top`.

### `to_input(pathfile, to='(-3,0,0)', width=8, height=8, name="temp")`
Insert an external image as input. `pathfile` is the image path (relative to the compile directory).

### `to_Conv(name, s_filer=256, n_filer=64, offset="(0,0,0)", to="(0,0,0)", width=1, height=40, depth=40, caption=" ")`
Convolution layer (Box). `s_filer` is shown on the z-axis label; `n_filer` on the x-axis label. `offset` is displacement relative to the `to` anchor.

### `to_ConvConvRelu(name, s_filer=256, n_filer=(64,64), offset="(0,0,0)", to="(0,0,0)", width=(2,2), height=40, depth=40, caption=" ")`
Double conv + ReLU (RightBandedBox). `n_filer` and `width` are pairs.

### `to_Pool(name, offset="(0,0,0)", to="(0,0,0)", width=1, height=32, depth=32, opacity=0.5, caption=" ")`
Pooling layer; red semi-transparent Box.

### `to_UnPool(name, offset="(0,0,0)", to="(0,0,0)", width=1, height=32, depth=32, opacity=0.5, caption=" ")`
Upsampling layer; blue semi-transparent Box.

### `to_ConvRes(name, s_filer=256, n_filer=64, offset="(0,0,0)", to="(0,0,0)", width=6, height=40, depth=40, opacity=0.2, caption=" ")`
Residual bypass (gray-white RightBandedBox) for skip-connection visualization.

### `to_ConvSoftMax(name, s_filer=40, offset="(0,0,0)", to="(0,0,0)", width=1, height=40, depth=40, caption=" ")`
Conv-style Softmax output (Box with zlabel only).

### `to_SoftMax(name, s_filer=10, offset="(0,0,0)", to="(0,0,0)", width=1.5, height=3, depth=25, opacity=0.8, caption=" ")`
Fully connected / Softmax output layer (vertical bar Box).

### `to_Sum(name, offset="(0,0,0)", to="(0,0,0)", radius=2.5, opacity=0.6)`
Addition node (Ball showing `$+$`).

## Connection Functions (tikzeng)

### `to_connection(of, to)`
Draw the main data-flow arrow from `{of}-east` to `{to}-west`. `of` and `to` are layer `name` values (without anchor suffixes).

### `to_skip(of, to, pos=1.25)`
Skip connection: route from the northeast corner of `of` over the top to the north side of `to`, using the `copyconnection` style (blue-green dashed arrow).

## Output Function (tikzeng)

### `to_generate(arch, pathname="file.tex")`
Write all strings in `arch` sequentially to `pathname`.

## Composite Blocks (blocks)

Return a list of layer fragments; append with `arch += block_xxx(...)` or `arch.extend(...)`.

### `block_2ConvPool(name, botton, top, s_filer=256, n_filer=64, offset="(1,0,0)", size=(32,32,3.5), opacity=0.5)`
ConvConvRelu → Pool, with connection `botton` → `ccr_{name}`. `size` is `(height, depth, width)`. Layer names: `ccr_{name}`, `top` (Pool).

### `block_Unconv(name, botton, top, s_filer=256, n_filer=64, offset="(1,0,0)", size=(32,32,3.5), opacity=0.5)`
Upsampling decoder block: UnPool → ConvRes → Conv → ConvRes → Conv, connecting `botton` → `unpool_{name}`. Final output layer name is `top`.

### `block_Res(num, name, botton, top, s_filer=256, n_filer=64, offset="(0,0,0)", size=(32,32,3.5), opacity=0.5)`
Chain of `num` Conv layers (intermediate layers `{name}_0`…`{name}_{num-2}`, final layer `top`), each with `to_connection`, plus `to_skip` between `layers[1]` and `layers[-2]`.

## Layout Conventions

- **Positioning**: `to="(prev-east)"` places a new layer east of the previous one; `offset="(dx,dy,dz)"` fine-tunes position (TikZ 3D coordinates).
- **Sizing**: `height` and `depth` control block height and depth (usually shrink as feature maps downsample); `width` controls block thickness.
- **Colors**: Controlled by macros in `to_cor()`; no Python-side color args needed (except fixed fill macros on Pool/Unpool layers).
- **projectpath**: Argument to `to_head` is the relative path from `PROJECT_DIR` to `SCRIPTS_DIR`.
